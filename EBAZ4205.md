# EBAZ4205（Zynq-7010）SD 启动

仓库根目录的原理图文件名和板上器件均对应 **EBAZ4205**，不是
EBAZ4250。本配置的 Yocto 机器名为 `ebaz4205-zynq7`。

## 硬件准备

1. 如果 TF 卡座未焊接，先补焊卡座及其周边缺件。
2. 将启动选择电阻从 `R2584` 移到 `R2577`，把启动模式从 NAND 改为 SD。
3. J7 是 3.3 V TTL 串口，参数为 115200、8N1。只连接 GND、板端 TX 到
   USB 串口 RX、板端 RX 到 USB 串口 TX；不要连接 USB 串口的 VCC。
4. 板卡通常从 DATA1/DATA2/DATA3 供电。若要从 J3/J4/J5 供电，先确认
   D24 已安装并确认极性。建议使用稳定的 12 V、至少 0.4 A 电源。

## 编译

从仓库根目录执行：

```sh
source ./edf-init-build-env build
bitbake edf-linux-disk-image
```

初始化脚本会自动把仓库内的 `meta-ebaz4205` 加入 `BBLAYERS`，并将新建构建
目录的默认 `MACHINE` 设为 `ebaz4205-zynq7`；这些规则本身也保存在版本库中。

缓存路径保存在受版本管理的
`meta-ebaz4205/conf/machine/ebaz4205-zynq7.conf` 中：

```bitbake
DL_DIR = "${HOME}/edf202601-cache/downloads"
SSTATE_DIR = "${HOME}/edf202601-cache/sstate-cache"
BB_NO_NETWORK = "1"
```

该机器配置为严格离线模式：不使用 EDF 的在线源码镜像或在线 sstate 镜像，
也不允许任何配方回退到网络。EBAZ4205 的 GPL `ps7_init_gpl.c/.h` 已从
`blockdesign_1_wrapper.xsa`（Vivado 2020.2，XSA SHA256：
`62cf063441290095a56b0229457f232de367aaa08207b5deb6061c73a86e1a39`）提取并
随 BSP 层保存到版本库中。它提供与已验证硬件设计一致的 DDR、PLL、MIO 和
外设初始化参数。开始编译前，必须先解压并准备好上面的 `downloads` 和
`sstate-cache` 目录；若其中缺少必需内容，BitBake 会直接报告 fetch 或
sstate miss（需要执行任务），不会联网补齐。

构建产物位于：

```text
build/tmp/deploy/images/ebaz4205-zynq7/
```

其中推荐直接烧写：

```text
edf-linux-disk-image-ebaz4205-zynq7.rootfs.wic.xz
```

实际文件名可能带版本/时间戳；同目录下通常有不带时间戳的符号链接。

## 烧写 SD 卡

先用 `lsblk -p` 确认整张卡的设备名。以下 `/dev/sdX` 必须替换成整张 SD
卡，不能写成分区（例如 `/dev/sdX1`）。写错设备会覆盖其他磁盘。

优先使用 bmaptool：

```sh
cd build/tmp/deploy/images/ebaz4205-zynq7
sudo umount /dev/sdX?* 2>/dev/null || true
sudo bmaptool copy \
  edf-linux-disk-image-ebaz4205-zynq7.rootfs.wic.xz \
  /dev/sdX
sync
```

也可以使用 `dd`：

```sh
xzcat edf-linux-disk-image-ebaz4205-zynq7.rootfs.wic.xz | \
  sudo dd of=/dev/sdX bs=16M status=progress conv=fsync
```

写入完成后安全弹出 SD 卡，插到断电的板卡上，再上电并通过 J7 查看启动
日志。EDF 镜像采用三分区布局：第 1 分区包含 `boot.bin`、`u-boot.img` 和
`boot.scr`，第 3 分区是 Linux 根文件系统。

## 早期启动诊断

当前 `boot.bin` 在正式 SPL 控制台和 DDR 可用前，会通过 UART1 输出以下阶段
标记：

```text
EBAZ4205 SPL: clock+UART OK
EBAZ4205 SPL: DDR init OK
EBAZ4205 SPL: PS7 init OK
```

如果只看到第一行，说明故障位于 DDR 初始化；如果一行都没有，则应优先检查
BootROM 是否从 SD 读到了 `boot.bin`、启动模式电阻、SD 卡第一分区以及串口
接线。若打印 `DDR init FAILED`，说明 XSA 参数已执行，但 DDR 状态轮询超时。

## 当前硬件范围

SD、256 MiB DDR3、UART1 和 NAND 已由配置描述。板载 IP101GA 以太网 PHY
通过 PL 的 EMIO/MII 连接，必须额外加载匹配的 FPGA bitstream 才能真正收发
数据。虽然该 XSA 内含 bitstream，本基础 SD 启动配置目前只采用其中的 PS7
初始化文件，没有将 bitstream 捆绑到启动镜像，因此以太网默认不可用。
